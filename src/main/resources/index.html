Markdown parser service.

Converts markdown-style text to html.

Available endpoints listed below:

 

1) Sign in - authorizes user if credentials are valid

Endpoint: /api/v1/signIn

Method: POST

Security header required: false

Body example: {"email": "test@test.com", "password": "pwd"}

Response: 200 if everything is ok. Response headers contains security token key

Response security header example: Api-Token: dfcf8120-3152-4b0e-8451-17700fc9ed94

 

2) Sign up - creates user if possible

Endpoint: /api/v1/signUp Method: POST

Security header required: false

Body example: {"email": "test@test.com", "pasword": "pwd"}

Response: 201 if user successfully created, otherwise - bad request

 

3) Logout - removes user's auth token from the storage

Endpoint: /api/v1/logout

Method: POST

Security header required: true

Body example: {}

Response: 200

 

4) Parse markdown - converts markdown to html

Endpoint: /api/v1/parseMarkdown

Method: Post

Security header required: true

Body example: {"source": "#####Loremipsum\nDolorsitamet,consetetur**sad**ip* scing *elitr,sed[diam](http://mysite.com)nonumyeirmodtempor"}

Response: 200 if parsed sucessfully. Bad request with cause otherwise

Response mime type: `text\html`
